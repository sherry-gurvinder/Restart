Creating Object Literals

In JavaScript, objects are a fundamental data type used to store collections of data in key-value pairs.

Objects can be assigned to variables just like any JavaScript type. We use curly braces, {}, to designate an object literal

let spaceship = {}; // spaceship is an empty object

Accessing Properties:

  • dot notation, .

  • Bracket Notation: The second way to access a key’s value is by using bracket notation, [ ].

Property Assignment

Objects are mutable meaning we can update them after we create them!

We can use either dot notation, ., or bracket notation, [], and the assignment operator, = to add new key-value pairs to an object or change an existing property.

  • If the property already exists on the object, whatever value it held before will be replaced with the newly assigned value. If there was no property with that name, a new property will be added to the object.

    It’s important to know that although we can’t reassign an object declared with const, we can still mutate it, meaning we can add new properties and change the properties that are there.

    const spaceship = {type: 'shuttle'}; spaceship = {type: 'alien'}; // TypeError: Assignment to constant variable. spaceship.type = 'alien'; // Changes the value of the type property spaceship.speed = 'Mach 5'; // Creates a new key of 'speed' with a value of 'Mach 5'

  • Delete: You can delete a property from an object with the delete operator.

    const spaceship = { 'Fuel Type': 'Turbo Fuel', homePlanet: 'Earth', mission: 'Explore the universe'}; delete spaceship.mission; // Removes the mission property delete spaceship.'Fuel Type'; // throw error

Defining Methods in Objects

When the data stored on an object is a function we call that a method. A property is what an object has, while a method is what an object does.

Do object methods seem familiar? That’s because you’ve been using them all along! For example console is a global JavaScript object and .log() is a method on that object. Math is also a global JavaScript object and .floor() is a method on it.

  • We can include methods in our object literals by creating ordinary, colon-separated key-value pairs. The key serves as our method’s name, while the value is an anonymous function expression.

    let object_name { function_Name () {}, function_Name () {}, function_Name () {}, function_Name () {}, }; object_Name.function_Name(); const alienShip = { invade: function () { console.log('Hello!') } };

  • With the new method syntax introduced in ES6 we can omit the colon and the function keyword.

    const alienShip = { invade () { console.log('Hello!') } };

  • Object methods are invoked by appending the object’s name with the dot operator followed by the method name and parentheses:

    alienShip.invade(); // Prints 'Hello!'

Nested Objects

You can define nested objects by including one object as a value for a key in another object:

  • Creating Nested Objects

    const person = { name: 'John Doe', age: 30, address: { street: '123 Main St', city: 'Anytown', country: 'USA' }, contact: { email: 'john.doe@example.com', phone: '555-1234' } }; console.log(person.address.city); // Output: Anytown console.log(person.contact.email); // Output: john.doe@example.com

  • Accessing Nested Object Properties

    console.log(person.address.city); // Output: Anytown console.log(person['contact']['phone']); // Output: 555-1234

  • Modifying Nested Object Properties

    // Update a nested property person.address.city = 'Newtown'; console.log(person.address.city); // Output: Newtown // Add a new property to a nested object person.address.zipCode = '12345'; console.log(person.address.zipCode); // Output: 12345

  • Adding Nested Objects

    person.social = { facebook: 'facebook.com/johndoe', twitter: 'twitter.com/johndoe' }; console.log(person.social.facebook); // Output: facebook.com/johndoe

  • Pass By Reference: In JavaScript, when you pass an object as a parameter to a function, you're passing a reference to that object, not a copy of it. Therefore, if you modify the object's properties within the function, the changes will be reflected in the original object.

    // Define an object const car = { make: 'Toyota', model: 'Camry', year: 2023 }; // Function that modifies the object carObj.year = 2024; // Modify the property of the object } // Call the Function updateCar(car); console.log(car.year); // Output: 2024

  • Looping Through Objects: use For.....IN

    const car = { make: 'Toyota', model: 'Camry', year: 2023 }; for (const key in car) { if (car.hasOwnProperty(key)) { console.log(`${key}: ${car[key]}`); } } // Output: // make: Toyota // model: Camry // year: 2023

Advanced Objects Introduction

'this' Keyword:

The this keyword references the calling object which provides access to the calling object’s properties.

  • Global Context:

    console.log(this); // In a browser, this logs the window object.

  • Object Method: When used inside a method of an object, this refers to the object itself.

    const person = {
    name: 'Gurvnder',
    greet: function() {
    console.log(this.name);
    }
    };
    person.greet(); // Logs "Gurvinder"

  • Constructor Function: In a constructor function (used with new), this refers to the newly created instance.

    function Person(name) {
    this.name = name;
    }
    const gurvinder = new Person('Gurvinder');
    console.log(gurvinder.name); // Logs "Gurvinder"

  • Arrow Functions: Arrow functions do not have their own this context; instead, they inherit this from their enclosing lexical context.

    const person = {
    name: 'Gurvinder',
    greet: function() {
    const innerFunc = () => {
    console.log(this.name); // 'this' refers to the person object
    };
    innerFunc();
    }
    };
    person.greet(); // Logs "Gurvinder"

  • Event Handlers: In event handlers, this refers to the element that triggered the event.

    document.querySelector('button').addEventListener('click', function() {
    console.log(this); // Logs the button element
    });

get and set

  • In JavaScript, getters are special methods in objects or classes that allow you to access properties in a controlled way. They are part of the broader concept of accessor properties, which also includes setters.

    const person = {
    firstName: 'Gurvinder',
    lastName: 'singh',
    // Getter to compute the full name
    get fullName() {
    return `${this.firstName} ${this.lastName}`;
    }
    };
    console.log(person.fullName); // "Gurvinder Singh"

  • setters are special methods that allow you to define custom behavior when setting the value of an object's property. They provide a way to intercept property assignments, making it possible to control how a property is updated or to perform additional actions whenever a property value changes.

    const person = {
    _name: '',
    set name(newName) {
    if (typeof newName === 'string' && newName.length > 0) {
    this._name = newName;
    } else {
    console.log('Please provide a valid name');
    }
    },
    get name() {
    return this._name;
    }
    };
    person.name = 'Gurvinder'; // Sets the name to 'Gurvinder'
    console.log(person.name); // Logs 'Gurvinder'
    person.name = ''; // Logs 'Please provide a valid name'
    console.log(person.name); // Still logs 'Gurvinder'

Factory Functions

A factory function in JavaScript is a function that returns a new object every time it is called.

A factory function is just a regular function that creates and returns a new object. You can use it to create multiple objects with the same structure.

Unlike constructors that use the new keyword, factory functions create objects without the new keyword. They are a popular way to create objects in JavaScript because they provide more flexibility and avoid some of the pitfalls associated with the this keyword in constructors.

  • Factory Function

    function createPerson(name, age) {
    return {
    name: name,
    age: age,
    greet() {
    console.log(`Hello, my name is ${this.name} and I am ${this.age} years old.`);
    }
    };
    }
    const person1 = createPerson('Gurvinder', 30);
    const person2 = createPerson('Simran', 25);
    person1.greet(); // Logs: "Hello, my name is Gurvinder and I am 30 years old."
    person2.greet(); // Logs: "Hello, my name is Simran and I am 25 years old."

  • Constructor Function:

    function Person(name, age) {
    this.name = name;
    this.age = age;
    }
    Person.prototype.greet = function() {
    console.log(`Hello, my name is ${this.name} and I am ${this.age} years old.`);
    };
    const person = new Person('Gurvinder', 30);
    person.greet(); // Logs: "Hello, my name is Gurvinder and I am 30 years old."

    Why Use Factory Functions?
    No new Keyword: You don't need to use new like you do with constructor functions. Just call the function, and it returns an object. No this Issues: Since factory functions don't rely on this, you avoid common mistakes where this gets misinterpreted. Flexible: You can easily add or change properties inside the factory function.

Destructured assignment

In JavaScript is a convenient way to extract values from objects or arrays and assign them to variables. It's especially useful when dealing with complex data structures.

Destructuring Objects

  • When you have an object, you can extract its properties into individual variables using destructuring.

    const person = {
    name: 'Gurvinder',
    age: 30,
    city: 'Toronto'
    };
    // Destructuring the object
    const { name, age, city } = person;
    console.log(name); // Logs: 'Gurvinder'
    console.log(age); // Logs: 30
    console.log(city); // Logs: 'Toronto'

    Here, { name, age, city } extracts the values from the person object and assigns them to variables with the same name as the object’s properties.

  • Renaming Variables:

    const person = {
    name: 'Gurvinder',
    age: 30,
    city: 'Toronto'
    };
    // Destructuring with renaming
    const { name: fullName, age: years, city: hometown } = person;
    console.log(fullName); // Logs: 'Gurvinder'
    console.log(years); // Logs: 30
    console.log(hometown); // Logs: 'Toronto'

Destructuring Arrays

  • Destructuring works similarly with arrays, allowing you to unpack values from an array into individual variables.

    const numbers = [10, 20, 30];
    // Destructuring the array
    const [first, second, third] = numbers;
    console.log(first); // Logs: 10
    console.log(second); // Logs: 20
    console.log(third); // Logs: 30

  • Skipping Elements: You can skip elements in an array by leaving an empty space in the destructuring assignment.

    const numbers = [10, 20, 30];
    // Skip the second element
    const [first, , third] = numbers;
    console.log(first); // Logs: 10
    console.log(third); // Logs: 30

  • Default Values: You can provide default values in case the property or element doesn’t exist:

    const person = {
    name: 'Gurvinder',
    age: 30
    };
    // Providing default values
    const { name, age, city = 'Unknown' } = person;
    console.log(city); // Logs: 'Unknown'

  • Nested Destructuring:If you have a nested object or array, you can destructure it similarly:

    const person = {
    name: 'Gurvinder',
    address: {
    city: 'Toronto',
    zip: 'M1t'
    }
    };
    // Nested destructuring
    const { name, address: { city, zip } } = person;
    console.log(city); // Logs: 'Toronto'
    console.log(zip); // Logs: 'M5V'
