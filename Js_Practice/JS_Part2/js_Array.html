Create an Array

In JavaScript, an array is a special type of object used to store ordered collections of values. These values can be of any type, including numbers, strings, objects, and even other arrays. Arrays are very versatile and provide many built-in methods to manipulate and interact with the data they contain.

Array Initialization and Creation

  • Array Literals: The most straightforward and commonly used way to create arrays is through array literals:

    let fruits = ['apple', 'banana', 'cherry'];

    let emptyArray = []; console.log(emptyArray); // Outputs: []

  • Array Constructor: Use the Array constructor when you specifically need to create an array of a particular length without initializing its values.

    Creating an empty array

    let emptyArray = new Array(); console.log(emptyArray); Outputs: []

    // Creating an array with a specific length (but uninitialized) let uninitializedArray = new Array(5); console.log(uninitializedArray); // Outputs: [ <5 empty items> ]

    // Creating an array with values let valuesArray = new Array('apple', 'banana', 'cherry'); console.log(valuesArray); // Outputs: ['apple', 'banana', 'cherry']

Dot Notation vs. Bracket Notation

Dot Notation (city.key): This is used to access properties of an object where the property name is a fixed identifier (e.g., city.school).

Bracket Notation (city[key]): This is used to access properties dynamically using a variable (e.g., city['school'] or city[key] where key is a variable).

Arrays with let and const

Variables declared with let can be reassigned. Variables declared with the const keyword cannot be reassigned

However, elements in an array declared with const remain mutable. Meaning that we can change the contents of a const array, but cannot reassign a new array or a different value.

re-assign an entire new array using the let/const keyword

  • Notice that you can re-assign elements in an array and re-assign an entire new array to a variable declared using the let keyword.

    let sampleArr = [1, 2, 3]; sampleArr = ['a string'];

  • Can not possible to re-assign a new array using const keyword

    const myList = ['car','bike', 'plane'] myList = ['trip to paris']; // this will throw a error

The .length property

Same like string we also have a built-in properties is length and it returns the number of items in the array. We access the .length property just like we do with strings.

When we want to know how many elements are in an array, we can access the .length property.

  • To access the length property, use dot notation like the example below:

    let fruit_Basket = ['Banana','Apple','Grapes','Melon']; console.log(fruit_Basket.length); // Output: 3

Nested Arrays

When an array contains another array it is known as a nested array.

  • To access the nested arrays we can use bracket notation with the index value,

    const nestedArr = [[1], [2, 3]]; console.log(nestedArr[1]); // Output: [2, 3] const nestedArr = [[1], [2, 3]]; console.log(nestedArr[1][1]); // Output: [3]// First subscript denoted the index of the array, second subscript gives you the value 3 is the element at index 1 of the sub-array [2, 3].

hasOwnProperty

The hasOwnProperty method is a built-in method in JavaScript that is used to check whether an object has a specific property as its own property (i.e., not inherited through the prototype chain).

  • object.hasOwnProperty(propertyName) object: The object on which the method is called. propertyName: The name of the property you want to check for. Returns true: If the object has the specified property as its own property. false: If the object does not have the specified property as its own property or if the property is inherited.

    let person = { name: "John", age: 30 }; console.log(person.hasOwnProperty("name")); // true console.log(person.hasOwnProperty("age")); // true console.log(person.hasOwnProperty("toString")); // false, because `toString` is inherited
